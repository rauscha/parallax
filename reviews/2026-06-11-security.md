## 1. Executive verdict

**Overall risk: LOW. Not a ship-blocker.**

Parallax is a self-contained static web app with no backend, no auth, no network egress of user data (share state lives in `location.hash`, never sent anywhere; presets/MIDI stay client-side). The trust boundaries are deliberately defensive: every share-URL field is coerced/clamped/dropped in `serialization.ts#decodeState`, the worklet message channel carries only self-originated integer-coerced values, there are **zero** `{@html}`/`innerHTML`/`eval`/`new Function` sinks, all text is Svelte-auto-escaped, the SVG notation renderer is built from typed numeric geometry (no string interpolation of user data), a sane CSP ships in the production build, `npm audit` is clean (0 vulns, 474 pkgs), the lockfile resolves entirely to `registry.npmjs.org` with integrity hashes, and the deploy workflow uses least-privilege permissions.

The only concrete findings are **two denial-of-service vectors reachable by a crafted share link** (an unbounded melody-event count, and an unbounded `position` field that blows up the staff renderer's ledger-line loop). Both require a victim to *open an attacker's link*, neither escapes the tab, neither corrupts other origins, and the fix for each is a one-line clamp. They are worth fixing before ship but do not block it.

There is one **process/supply-chain note** (carried over from the 2026-05-31 review): the prebuilt WASM/JS engine artifacts in `public/` are committed binaries that CI ships without rebuilding from source. That is an accepted, documented trade-off, but the provenance should be pinned (see M-3).

## 2. Findings by severity

### MEDIUM

#### M-1 â€” Unbounded `position` field â†’ renderer hang (DoS via share link)
**Files:** `src/state/serialization.ts:122-123`, `src/notation/render.ts:269-279`, `src/notation/StaffEditor.svelte:131`

`decodeEvent` accepts a melody event's optional staff `position` with only a finiteness check:

```ts
if (position != null && Number.isFinite(Number(position))) {
  ev.position = Math.round(Number(position));   // no range clamp
}
```

Unlike `midi` (clamped 0â€“127) and `startStep` (clamped 0â€“63), `position` has **no bounds**. The staff renderer feeds it straight into `ledgersFor(position)` (`StaffEditor.svelte:131`), whose loop runs once per half-step between the staff and the note:

```ts
for (let p = TOP_LINE_POS + 2; p <= position; p += 2) out.push(p);
```

**Exploit:** a share URL whose first event is `[0,4,60, 2000000000, 0]` (well under the URL length the lz-string layer produces â€” ~125 chars total) decodes to a valid event with `position = 2e9`. On render, `ledgersFor` loops ~1e9 times allocating an array of a billion entries â†’ the tab freezes / OOM-crashes. Measured: `position = 200000` already yields ~100k array entries in a few ms; scale that to 2e9 and the tab is gone. The malicious event is also written into `melodyStore`, so if the victim saves it as a preset the hang re-triggers on every load.

This is real (not theoretical) but **requires the victim to open an attacker-supplied link**, and the blast radius is a single tab on a single origin â€” no data exfiltration, no cross-origin impact.

**Fix:** clamp `position` to the staff's sane display range in `decodeEvent`, e.g. `Math.max(-40, Math.min(40, Math.round(Number(position))))` (covers C1â€¦C8 with headroom). Optionally also guard `ledgersFor` to bail past a hard cap.

#### M-2 â€” Unbounded melody-event count â†’ decompression-amplified DoS
**Files:** `src/state/share-url.ts:32`, `src/state/serialization.ts:162-164`

`decodeState` maps over `s.ev` with no cap on array length; events that pass the per-event checks (e.g. `[0,4,60,null,0]`, all valid) are all kept. lz-string is a high-ratio compressor over repetitive JSON: measured **8 MB of JSON (500,000 identical events) compresses to a ~35 KB URL â€” a 230Ă— amplification â€” and decompresses in 24 ms.** A 35 KB hash is well within what browsers accept.

**Exploit:** a crafted `#p=â€¦` link decodes to a 500k-event `melodyStore`. The Tone.js `Part` rebuild, the StaffEditor `#each previewEvents`, and `silentGaps`/`fillRestGap` then iterate/sort that array on the main thread â†’ multi-second-to-permanent UI freeze. Persisted into IndexedDB if saved.

Same caveats as M-1: requires opening a hostile link; tab-local only.

**Fix:** cap events in `decodeState`, e.g. `s.ev.slice(0, 256).map(decodeEvent)`. 64 steps monophonic means a legitimate melody never exceeds ~64 events, so a generous cap (256) is invisible to real use. Consider also de-duplicating by `startStep` since the grid is monophonic.

### LOW

#### L-1 â€” `durationSteps` has no upper bound
**File:** `src/state/serialization.ts:117`

`decodeEvent` rejects `dur < 1` but accepts any large finite duration. Downstream this is mostly clamped (`fillRestGap`/`silentGaps` cap at `TOTAL_STEPS`; `convert.ts` re-clamps on MIDI export), and `durationSpan = durationSteps * stepWidth` just produces a wide (off-screen) tail rather than a loop â€” so impact is cosmetic, not a hang. Worth clamping to `TOTAL_STEPS` for tidiness when fixing M-1/M-2.

#### L-2 â€” `sanitizeParams` keeps unbounded / arbitrary-key params
**File:** `src/state/serialization.ts:95-104`

Decoded patch params accept any finite numeric value under any string key. The engine layer is the backstop and it is solid: `BraidsEngine.setParameter` coerces with `| 0` / clamps (e.g. model index `Math.max(0, Math.min(46, index|0))`, envelope clamps at `render.ts`-adjacent code), and the worklet `parameterDescriptors` enforce `minValue`/`maxValue` on the AudioParams. Unknown keys are simply ignored by the `switch`. So a hostile param value can't reach the DSP out of range and can't pollute anything. No prototype-pollution risk: keys like `__proto__` land as own-enumerable string keys on a fresh object literal via `out[k] = â€¦`, not on the prototype, and are never used as lookup keys against a shared object. **Informational** â€” current behavior is safe; tightening to a per-engine allow-list would be defense-in-depth, not a fix.

#### L-3 â€” No `frame-ancestors` / clickjacking header
**File:** `vite.config.ts:27-28` (acknowledged in-code)

`frame-ancestors` and `X-Frame-Options` can't be set via `<meta>`, and GitHub Pages can't serve custom headers, so the app can be framed by any site. For a no-auth, no-secrets synth toy the clickjacking impact is negligible (there's nothing sensitive to trick a click into). Documented limitation; accept for v1.

### INFORMATIONAL / Supply chain

#### M-3 (note) â€” Committed prebuilt WASM/JS engine artifacts are unauditable in CI
**Files:** `public/braids.wasm`, `public/braids.js`, `public/braids-worklet.js`, `public/plaits.wasm`, `public/plaits.js`, `public/plaits-worklet.js`, `public/laxsynth-worklet.js`

Per the locked decision (CLAUDE.md), CI ships these committed binaries without an Emscripten toolchain, so `deploy.yml` never rebuilds them from `dsp/shim/*.cc`. This is a reasonable, deliberate trade-off (no toolchain in CI), but it means the binary that reaches users is trusted on the committer's word, not reproduced from source by the pipeline. The worklets themselves are hand-maintained plain JS (readable, reviewed: integer-clamped param descriptors, `clamp01` helpers, no network, no eval) â€” only the `.wasm`/Emscripten `.js` glue are opaque.

**Recommendation (not a blocker):** commit a short provenance record â€” source commit hash of `pichenettes/eurorack`, emcc version, and a SHA-256 of each artifact â€” in `dsp/shim/` or a `PROVENANCE.md`, so a future rebuild can be checked for reproducibility. This is the one place where "trust" isn't backed by "verify."

## 3. Things checked and found clean

- **DOM injection sinks:** grepped the whole `src/` tree â€” **no** `{@html}`, `innerHTML`, `outerHTML`, `insertAdjacentHTML`, `document.write`, `eval`, or `new Function`. All dynamic text (preset names, model names, key/scale labels) flows through Svelte interpolation, which auto-escapes.
- **SVG notation renderer:** `render.ts` and `StaffEditor.svelte` build the staff from typed numeric geometry; glyphs come from a fixed `GLYPH` table of Bravura codepoints, not user strings. No path/attribute is concatenated from share-URL text. (The DoS in M-1 is a numeric-loop issue, not injection.)
- **Postcard renderer (`postcard.ts`, `PostcardModal.svelte`):** pure `<canvas>` 2D drawing. User strings (model name, key) go to `ctx.fillText`, which cannot execute markup. Values clamped (`Math.max(0,Math.min(1,value))` on dials, piano-roll widths floored/min'd). Export is `canvas.toBlob` â†’ object URL â†’ download; copy is `navigator.clipboard.write` of an image blob. No injection surface.
- **MIDI import (`io.ts`, `convert.ts`):** parsed by `@tonejs/midi` from an `ArrayBuffer`; `midiToMelody` is bounded by construction â€” a `Map` keyed on step collapses to â‰¤64 entries regardless of input size, notes outside step/pitch range are dropped and counted, durations clamped â‰Ą1 and trimmed to the next note. A malformed or huge `.mid` can't produce an unbounded melody. (`@tonejs/midi`'s own parser is the trust boundary for the raw byte parse; no known CVE, audit clean.)
- **Share-URL pipeline structurally:** `readSharedState` â†’ `decompressFromEncodedURIComponent` (returns `null`/`""` on garbage, handled) â†’ `JSON.parse` in try/catch â†’ `decodeState` rejects wrong `version`, missing sections, bad types, returns `null` cleanly. No throw escapes to the caller. lz-string itself has no code-execution path (pure string transform).
- **Prototype pollution:** decode builds fresh object literals and assigns own keys; no recursive merge, no `Object.assign` from untrusted source into a shared target, no dynamic property lookup against a prototype-bearing object. Safe.
- **IndexedDB / localStorage trust on load:** `persistence.ts` filters corrupt preset rows (`listPresets`), `loadPreset` runs the same defensive `decodeState`. `editorMode.ts` reads are all allow-listed (`"-1"|else`, `"grid"|"staff"`) or `parseInt`-clamped to a numeric range. No trusted-as-code or trusted-as-HTML read.
- **CSP (production):** verified present in built `dist/index.html`. `default-src 'self'`, `script-src 'self' 'wasm-unsafe-eval'` (no `unsafe-inline`/`unsafe-eval`), `object-src 'none'`, `base-uri 'self'`, `form-action 'none'`, `connect-src 'self'`, `worker-src 'self' blob:` (blob justified for Tone's ticker; documented). `style-src 'unsafe-inline'` is required by Svelte inline style bindings â€” standard and low-risk. WASM/worklet/font needs all covered. Sound for a static app.
- **PWA / service worker:** Workbox precache, `registerType: autoUpdate`, manual `injectRegister: false` (no inline script for CSP). SW scope is the app base path (`/parallax/`), same-origin only; `controllerchange` reload is guarded against loops/first-install. No cache-poisoning surface beyond what an attacker who already controls the origin would have. `cleanupOutdatedCaches: true`.
- **Web MIDI (`midi-input.ts`):** requests access with `sysex: false` (no system-exclusive). Incoming messages are masked/bounds-read (`data[0] & 0xf0`, `data[1] ?? 0`) and only note-on/off + sustain CC are acted on; everything else ignored. Panic-on-blur/visibility-hidden prevents stuck notes. No way for a hostile device to do more than play notes.
- **Deploy workflow (`.github/workflows/deploy.yml`):** least-privilege `permissions:` (`contents: read`, `pages: write`, `id-token: write`), concurrency-guarded, official `actions/*` at major-version tags. No third-party actions, no `pull_request_target`, no secret exposure.
- **Supply chain:** `npm audit` (prod and full) â†’ **0 vulnerabilities** across 474 packages. Lockfile `resolved` URLs all point to `registry.npmjs.org`; 474 `integrity` hashes present. Dependency set is small and reputable (tone, @tonejs/midi, @tonaljs/tonal, lz-string, idb-keyval, nanostores). No source maps shipped to prod (default Vite build).
- **Secrets:** no API keys/tokens in source or build (no backend exists to hold credentials for).
- **Type-check:** `npm run check` passes clean (0 errors, 0 warnings).
