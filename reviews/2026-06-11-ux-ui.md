# Parallax — UX/UI Design Review

**Date:** 2026-06-11 · **Branch:** `claude/codebase-shipping-review-q5whpr` · **Method:** code-based review (no browser available — every Svelte component, theme file, and `index.html` read; layouts and flows reconstructed from markup + CSS; `npm run build` confirmed green, 19 precache entries). Items that genuinely need pixels are flagged in §6.

---

## 1. Verdict

**Grade: A−** · **Character sketch: tactile, literate, luminous.**

**Ship-readiness: yes — ship after the Sandbox contrast fixes (the one real blocker) and ideally the two small focus/zoom fixes.** This is one of the most carefully reasoned small-app UIs I've reviewed: the token system is a real design system (it parametrizes *behavior*, not just color), micro-interactions are engineered rather than sprinkled on, and nearly every CSS decision carries a comment explaining the failure it prevents. The prior review's P1 punch list (missing tokens, no keyboard nav in the picker, fake scope glow, contrast misses, parked Knob) has been comprehensively paid down. What remains is a short, specific list: Sandbox theme text-on-signal contrast actually fails the project's own stated AA rule, there is no undo anywhere despite three one-tap melody/patch-destroying actions, and modal focus management is absent.

## 2. What's excellent (specific)

- **The behavioral token system.** `tokens.css` doesn't just swap palettes — each theme tunes the oscilloscope's *physics*: `--scope-persist` (0.85 / 0.78 / 0.92), `--scope-bloom` (0.45 / 1.0 / 1.7), `--scope-trace-w` (`src/ui/themes/tokens.css:42-44, 100-102, 140-142`), consumed live by the canvas via `getComputedStyle` (`src/viz/Oscilloscope.svelte:85-94`). Lab gets a crisp vector trace, Sandbox a warm mid glow, Phosphor heavy CRT persistence. Typography identity is tokenized too: `--label-case: lowercase` for the TE-style Sandbox, `uppercase` + 0.08em tracking for Phosphor (`tokens.css:109-110, 152-153`). That's a design system, not three coats of paint.
- **The Knob is a small masterpiece.** Magnetic detent at factory default with catch/release *hysteresis* so it can't chatter, haptic tick only on catch, detent tick mark that lights while snapped (`src/ui/Knob.svelte:22-24, 99-110, 213, 274-277`); RAF-coalesced `onchange` with a guaranteed flush on release so the audio thread isn't flooded (`Knob.svelte:72-90`); full keyboard model — arrows, Shift×10, Home/End, Delete-to-default (`138-155`); proper `role="slider"` with `aria-valuetext` (`191-197`). Pre-holding the detent when you grab a knob already at default so it doesn't fire a spurious haptic (`118-120`) is the kind of detail almost nobody writes.
- **The knob ↔ Explain-card highlight channel.** Hover/focus/drag on a knob lights its explanation card and vice versa via a shared `activeParamStore` (`Knob.svelte:35-47`, `src/ui/ExplainPanel.svelte:56-66`), and the "Show me" sweep rides the same channel (`src/state/show-me.ts:179-181`). This makes the app's core teaching premise *spatial* — the words and the control find each other.
- **"Show me" sweep design** (`show-me.ts`): eased original → min → max → original path with the min→max leg longest ("that's the reveal", `:66-73`), rides existing audio if the transport is running, restores cleanly on blur/model-swap/user-grab. Thoughtful interruption semantics, centrally handled.
- **First-run choreography.** TapToStart explains *why* the gate exists ("Browsers require a click before they'll make sound. One tap and you're in.", `src/ui/TapToStart.svelte:54-56`), plays an A440 confirmation strike (`:34-35`), and the scope answers with a 700ms boot sweep timed to outlast the tone (`src/viz/Oscilloscope.svelte:16-23, 196-215`). The instrument *announces it's alive*. Desktop users can make sound in well under 10 seconds (QWERTY map is always armed); mobile gets the NoteStrip immediately.
- **The idle scope is never dead.** Breathing baseline + a faint beam dot sweeping every ~6s, both alpha-only and both gated behind `prefers-reduced-motion` (`Oscilloscope.svelte:134-160`). Direct, correct response to the prior review's "dead flat line" critique.
- **Mobile engineering with receipts.** The ≤720px layout comments narrate every trap they escaped — the flexbox `min-height:auto` scroll trap, the crushed-region bug, the grid `1fr`-rows-collapse bug (`src/App.svelte:501-558`, `src/notation/GridEditor.svelte:924-955`). Safe-area insets on top bar and transport (`App.svelte:550, 555-557`), 16px tempo input to defeat iOS focus-zoom (`:389-393`), long-press callout suppression on the NoteStrip (`src/ui/NoteStrip.svelte:192-198`), lazy pointer-capture on bar-swipe so taps still click (`GridEditor.svelte:501-513`). `pointer: coarse` blocks bump targets to 36–48px in *every* component.
- **Colorblind discipline as a stated invariant, mostly honored.** Audio status uses glyph shape ●/○ + case + weight, same color both states (`App.svelte:487-499`); selected model rows get fill + bold + `▸` marker + left edge bar "defence in depth" (`src/ui/ModelPicker.svelte:342-359`); dragging knobs get a surface lift, not just hue (`Knob.svelte:281-285`).
- **Copy voice.** "Search models — try 'bell', 'fm', 'noise'…" (`ModelPicker.svelte:179`), "Lo-fi (the Braids crunch)" (`src/ui/ParamPanel.svelte:43`), "Works best on a solo or exposed passage" (`src/ui/MatchPanel.svelte:319`), import honesty: "Imported 14 notes · 3 dropped (past bar 4 / overlaps)" (`src/ui/MidiMenu.svelte:52-55`). Plain, warm, precise.
- **Error/empty/loading states exist almost everywhere they should**: TapToStart error with `role="alert"` (`TapToStart.svelte:51-53`), engine-swap "Loading…" + inline error (`src/ui/EnginePicker.svelte:44-47`), empty-melody hint + "Load demo" (`App.svelte:140-142`), empty preset list (`src/ui/PresetMenu.svelte:137-138`), "No models match" (`ModelPicker.svelte:205-207`), Web MIDI unsupported message (`src/ui/MidiInput.svelte:33-34`).

## 3. UX issues, prioritized

### Blocker

- **B1 — Sandbox theme: text on `--signal` fails contrast, violating the project's own AA rule.** Computed: `--bg` #E9E5DC on `--signal` #FF5A1F = **2.48:1**; `--signal` as text on `--bg` = **2.48:1**, on `--surface-sunken` = **1.87:1**. The tokens file's own comment ("big pill code (>3:1 OK for graphics)", `tokens.css:87`) is arithmetically wrong — it's *below* 3:1 even by the graphics standard. Affected as **text**, mostly at 0.6–0.72rem (small-text 4.5:1 applies):
  - Every "active = signal fill + `color: var(--bg)`" button: Surprise (`App.svelte:233-246`), active PLAY (`:350-354`), active SCOPE/SPECTRUM (`:295-299`), active Staff/Grid (`:429-433`), TapToStart CTA (`TapToStart.svelte:86-97`), MIDI Enable (`MidiInput.svelte:76-89`), In Key/Chromatic active (`GridEditor.svelte:897-901`), staff tool active (`src/notation/StaffToolbar.svelte:95-98`), Save (`PresetMenu.svelte:205-218`), Choose file/Apply (`MatchPanel.svelte:503-515, 651-665`).
  - `color: var(--signal)` used directly as text: Explain panel model code on sunken surface — **1.87:1** (`ExplainPanel.svelte:141-150`) and the live `%` readouts (`:234-238`); grid root-row pitch labels (`GridEditor.svelte:762`); Match panel detected values and model code (`MatchPanel.svelte:594-600, 641-645`).
  - **Fix (small):** add an `--on-signal` token per theme (`lab`/`phosphor`: `var(--bg)` as today; `sandbox`: `#1A1A1A` — computes 5.58:1 on the orange) and switch all `color: var(--bg)`-on-signal sites to it; replace text-role `var(--signal)` with the already-provided `var(--signal-ink)` (5.25:1), which most components (e.g. `ModelPicker.svelte:251`) already use correctly. Lab (8.7:1) and Phosphor (15.6:1) are unaffected.

### Major

- **M1 — No undo, and three one-tap destructive actions.** "Surprise me" replaces engine, model, *all* params, key, scale, and the entire melody (`src/state/surprise.ts:56-87`) with no confirmation and no way back. Grid "⚄ Randomize" replaces the melody (`GridEditor.svelte:425-428`); "Clear" wipes it (`App.svelte:145`); MIDI import replaces events + tempo (`MidiMenu.svelte:50`). A user who spent 10 minutes hand-placing a melody loses it to one mis-tap on the dice. Presets at least get a native `confirm()` for overwrite/delete (`PresetMenu.svelte:59, 86`). **Fix:** a single-slot "snapshot before destructive action + toast with Undo button" covers all four call sites cheaply (state is already fully serializable — `share-url.ts` proves it). That's far better than confirmation dialogs, which would tax the playful "roll again" loop.
- **M2 — Staff editor is pointer-only; keyboard and screen-reader users get nothing.** `role="application"` with a gesture-only label (`src/notation/StaffEditor.svelte:522-529`), no `tabindex`, no key handlers. The Grid surface, by contrast, has a full cursor model — arrows/Space/Delete (`GridEditor.svelte:430-489, 595-608`). Since both edit the same melody, the floor fix is cheap: note in the staff's `aria-label` that the Grid surface offers keyboard editing, and make the Staff/Grid toggle the documented path. Better: port the Grid's cursor to the staff later.
- **M3 — No focus management on any modal/popover.** PostcardModal (`PostcardModal.svelte:127`), MatchPanel (`MatchPanel.svelte:279`), Preset/MIDI/Tools popovers: none move focus in on open, trap Tab, or restore focus on close. `aria-modal="true"` is claimed while background content stays fully tabbable — for a keyboard or SR user the dialog effectively isn't there. Escape *is* wired everywhere (good). **Fix:** focus the dialog (or first control) on open, restore the invoker on close; a ~15-line `trapFocus` action shared by all five surfaces.
- **M4 — iOS input auto-zoom fixed for tempo but not the other text inputs.** The comment at `App.svelte:389-392` shows the team knows the 16px rule, but the model search is 0.85rem (~13.6px) on coarse pointers (`ModelPicker.svelte:389-393`) and the preset name input is 0.74rem with no coarse override at all (`PresetMenu.svelte:193-204`). Focusing either on an iPhone zooms the whole PWA. **Fix:** `font-size: 16px` in their `(pointer: coarse)` blocks.
- **M5 — On phones, the hero action is hidden behind "⋯".** Surprise me — styled as "the playful hero action" (`App.svelte:232`) — collapses into the ToolsMenu overflow on ≤720px along with Share/Presets/MIDI (`src/ui/ToolsMenu.svelte:59-113`). A first-time phone user sees brand + ⋯ and nothing inviting. **Fix:** keep the ⚄ button (icon-only if needed) visible in the phone top bar; overflow the rest.

### Minor

- **m1 — Knob: `preventDefault()` on pointerdown (`Knob.svelte:121`) suppresses click-to-focus**, so after a mouse drag the arrow keys don't address that knob, and the engaged glow drops on release. Tab-focus still works. Consider `el.focus()` inside `onPointerDown`.
- **m2 — Deliberate `.blur()` after activating SCOPE/SPECTRUM (`App.svelte:77-79`) and after picking a model (`ModelPicker.svelte:93`)** hands the keys back to note-playing — defensible for an instrument, but it strands keyboard-only users mid-flow with no focus. At minimum exempt `:focus-visible` (keyboard) activations.
- **m3 — No semantic danger/error token.** Errors are styled `color: var(--accent)` (`EnginePicker.svelte:98`, `MatchPanel.svelte:522`, `PresetMenu.svelte:220`) — amber on Phosphor, gold on Lab, but plain **charcoal** on Sandbox (`--accent: #1A1A1A`), where an error message is visually indistinguishable from body text. Add `--danger` per theme.
- **m4 — Native `confirm()` in PresetMenu (`:59, 86`)** breaks theme immersion and shows browser chrome; fine functionally, inconsistent with the otherwise bespoke surfaces.
- **m5 — Long-press delete on the staff (500ms, `StaffEditor.svelte:151-160`)** has no in-progress affordance (no shrink/ring/progress) — the note just vanishes at the timer. Hint text covers discoverability only while the melody is empty (`App.svelte:140-141`).
- **m6 — Disabled PLAY (no notes) has no explanation** beyond the empty-state hint above it (`App.svelte:159`); a `title="Add notes to the sequencer first"` is free.
- **m7 — MatchPanel hint "Play notes with Z–M" (`MatchPanel.svelte:336`)** is desktop-only language shown on phones, where Z–M doesn't exist (and the NoteStrip is hidden under the overlay).
- **m8 — Knob affordances (`title` at `Knob.svelte:198`: drag/Shift/double-click)** are mouse-only discoverable; touch users never learn double-tap… which isn't even wired (only `ondblclick`). Consider the Explain panel teaching this once.
- **m9 — Landscape-notch safe areas:** only top/bottom insets are handled (`App.svelte:550, 556`); `env(safe-area-inset-left/right)` are not. Edge content can sit under the notch in landscape PWA. Verify on device.

## 4. Accessibility audit

**The honest summary: well above typical synth-toy baseline, with three structural gaps (M2, M3 above, and live-region silence).**

What's genuinely done right:
- **Reduced motion:** global transition/animation kill (`base.css:87-92`), plus *manual* gating in canvas land — idle breathing flattens, beam dot and boot sweep skip (`Oscilloscope.svelte:26-28, 137-139, 149, 196`), model pulse opts out (`ModelPicker.svelte:243-245`), toast static (`PwaToast.svelte:51-53`). Canvas animations are where reduced-motion is usually forgotten; here it isn't.
- **Focus visibility:** global `:focus-visible` 2px signal outline (`base.css:94-99`); Knob replicates it via box-shadow (`Knob.svelte:236-238`); grid surface has its own ring (`GridEditor.svelte:782-785`).
- **ARIA structure:** labeled regions/groups throughout (`App.svelte:74, 88, 101, 106`), `aria-pressed` on toggles, `aria-current` on selections, `aria-expanded/haspopup` on popovers, `role="alert"`/`role="status"` on errors/status lines (`MidiMenu.svelte:91`, `PostcardModal.svelte:144`), per-key `aria-label`s on the NoteStrip (`NoteStrip.svelte:135, 149`), proper `role="radiogroup"` on staff tools (`StaffToolbar.svelte:21-50`).
- **Contrast bookkeeping in the tokens file** — `--text-dim` values carry their measured ratios (`tokens.css:25, 85, 126`); I verified: lab 5.53:1, sandbox 5.49:1, phosphor 6.40:1 on their backgrounds. The discipline is real; the Sandbox `--signal` blind spot (B1) is the one place it lapsed.
- **`color-scheme` per theme** (`tokens.css:71, 112, 155`), so native selects/scrollbars match.

Gaps, in priority order:
1. **Dialogs don't manage focus** (M3). The `aria-modal` claim is currently false in practice.
2. **Staff editor invisible to AT** (M2); the Grid's keyboard model partially compensates but isn't announced anywhere.
3. **No live announcements for state the eye gets for free:** model changes (the pulse is visual-only), grid cursor position/note placement, Surprise results, octave shifts. A single polite `aria-live` status line (e.g. in the transport bar) announcing "CSAW · 02/47", "Note C4 placed, bar 2 beat 3", "Octave 5" would transform SR usability for modest cost.
4. **What a screen reader experiences today:** boots into a labeled dialog with one clear button (good); can choose engines/models (list buttons with `aria-current`), tweak every parameter (real sliders with value text — genuinely good), play notes via QWERTY, run transport — but cannot *enter* a melody on the staff, gets no feedback placing notes on the grid (cells are unlabeled `div`s; the cursor is visual-only), and never hears errors that appear outside `role="alert"` regions. Audio itself is, of course, the feedback channel for much of this — fair for a synth — but note entry feedback is silent when the transport is running (`GridEditor.svelte:135-137`).
5. Small: Oscilloscope/Spectrum canvases have `aria-label` but no `role="img"`; NoteStrip held keys lack `aria-pressed`; `viz-toggle` buttons blur themselves (m2).

## 5. Craft & beauty, per theme

**System-level:** radius scale (2/4/8) used with `calc()` nesting discipline (`ModelPicker.svelte:291`, inner pills = outer − padding — correct optical nesting); a real motion vocabulary (`--t-fast/--t-med/--t-spring`, the spring curve `cubic-bezier(0.34,1.56,0.64,1)` reused for the model pulse); hairline language (`--hairline` vs `--hairline-soft`) consistently separating structure from texture; the 1px-gap `background: var(--hairline)` grid seam trick (`App.svelte:253-258`). The fractal-noise grain layer with `mix-blend-mode: overlay` — one neutral tile serving both dark Lab and light Sandbox, with Phosphor opting out because "its scanlines are already the texture" and mobile opting out with a written rationale (`base.css:101-123`) — is the single best paragraph of CSS reasoning in the codebase.

**Phosphor (Braids — the landing theme): fully committed.** Mono type *everywhere* (`--font-heading` and `--font-sans` both JetBrains Mono, `tokens.css:149-150`), uppercase tracking, near-black `#0A0805` with warm-brown hairlines (`#2C2418` — aged bakelite, not neutral gray: a subtle, excellent choice), heaviest trace persistence (0.92), 1.7× bloom, scanlines via multiply blend (`tokens.css:159-171`), and a vignette that only Phosphor renders to "sell the curved-glass tube" (`Oscilloscope.svelte:261-268`). Amber `#FFB347` as the secondary signal is exactly right for the period. This theme reads as a CRT instrument *in the code*; verify the scanline pitch (2px/1px at 3px period) doesn't moiré on non-integer-DPR screens.

**Sandbox (Plaits): the personality is right, the execution has the one hole.** Warm putty body, dark inset screen (`--scope-bg: #0D0D0D` against `#E9E5DC` — the classic TE "dark glass in cream plastic" move), lowercase labels, charcoal-as-accent, dark knob pointer fixed from the prior review (`tokens.css:106`). But it's the only light theme, and it's where every `--signal`-as-text and text-on-signal decision breaks (B1). Also the only theme where the heavy `rgba(0,0,0,…)` drop shadows on popovers (`PresetMenu.svelte:187`) will read harsher — verify visually.

**Lab (Laxsynth): quietest of the three — competent, slightly under-charactered.** The "whisper of lavender" neutral grays + teal/gold are tasteful and AA-clean (8.7:1 signal), and "crisp vector trace, minimal halo" (`--scope-bloom: 0.45`) is a smart SNES-era read. But nothing here *says* SNES the way scanlines say CRT or the cream body says TE — no chunky bevel, no pixel-y accent, no D-pad-era detail. It reads "nice dark app theme." Acceptable to ship; the bit isn't fully committed.

**Craft slips (named):**
- **No spacing or type scale tokens.** Padding/gaps are raw px everywhere (`5px 12px`, `8px 10px`, gaps 4/6/8/10/12/14) and font sizes are a 13-step rem soup (0.58, 0.6, 0.62, 0.65, 0.66, 0.68, 0.7, 0.72, 0.74, 0.78, 0.8…) re-declared per component. It's *visually* consistent because one author was careful, but it's convention, not system — the first contributor (or AI session) without the house feel will drift. A `--space-1..6` and 4-step `--fs-*` scale would lock it in.
- **Shadow values hardcoded and inconsistent:** `0 14px 40px rgba(0,0,0,0.36)` (popovers), `0 24px 64px @ 0.4/0.45/0.5` (three different modal opacities — `TapToStart.svelte:75`, `PostcardModal.svelte:175`, `MatchPanel.svelte:451`), `0 10px 30px @ 0.4` (toast). One `--shadow-pop`/`--shadow-modal` pair would end it.
- **Stale palette ghosts:** canvas/postcard fallback colors reference a retired teal `#34E1C4` scheme (`Oscilloscope.svelte:87-90`, `PostcardModal.svelte:27-36`), and the PWA manifest `theme_color`/`background_color` are `#0B0E11` (`vite.config.ts:85-86`) — a color no current theme uses; the splash screen on installed Android won't match any skin (runtime `theme.ts` fixes the meta tag, not the manifest).
- **The duplicated `io-btn` style** — same ~20 lines in `PatchToolbar.svelte:67-88` and `PresetMenu.svelte:158-174`/`MidiMenu.svelte:99-114`; they've already drifted (one transitions `color`, one doesn't).
- **`--radius-lg` (8px) is used by exactly three surfaces** (the three modals); everything else is sm/md — fine, but it suggests the scale wasn't fully pressure-tested.

**Beautiful decisions worth naming:** the engine *is* the theme (selecting a voice re-skins the instrument and even the Android status bar via `theme.ts` — identity and sound fused); the detent tick that lights while you're held in it; the model code-pill spring pulse "because cycling models is the whole point of Braids" (`ModelPicker.svelte:233-242`); the boot sweep synced to the A440 strike; the postcard rendered in the live theme's own colors (`PostcardModal.svelte:23-37`).

## 6. Verify visually before ship

1. **Sandbox theme, after the B1 fix:** every active/filled button readable; Explain `%` readouts and model code legible on the sunken cards; grid root-row labels readable.
2. **Phosphor scanlines + grain interaction:** scanline period at DPR 1.25/1.5 (moiré risk); vignette darkness over the spectrum bars; bloom 1.7 not blowing out thick traces on a 4K display.
3. **iPhone (installed PWA):** focus model search and preset name inputs — confirm no page zoom after the M4 fix; safe-area in landscape (m9); NoteStrip chips vs. home-indicator gesture conflicts.
4. **Small-laptop heights (~700–768px):** the desktop grid is `overflow:hidden` with fixed fr rows (`App.svelte:261-268`) — confirm the Controls column scrolls rather than clips knob banks for Plaits (most knobs), and the 4-bar staff stays legible at its allotted 1.6fr.
5. **720px boundary:** resize across it with the ToolsMenu open (popover ⇄ `display:contents` flip mid-interaction); top bar with all six buttons at 721–900px widths — does it wrap?
6. **Theme transition moment:** swap engines and watch the full-page recolor — is the instant flip pleasing or jarring? (No transition is currently defined on theme change; a brief root-level color transition might sell it, or might smear — needs eyes.)
7. **Offline (installed PWA, airplane mode):** Google Fonts are *not* in the Workbox precache (`vite.config.ts:100-104` globs local files only; no runtime caching rule) — so offline, Lab/Sandbox fall back to system fonts and Phosphor to a system mono, despite the "Ready to work offline" toast. Decide: self-host the three families (best; Bravura already is) or add a Workbox runtime cache for `fonts.gstatic.com`.
8. **Long-press delete feel** on a real phone: 500ms against typical scroll intent; confirm Android's context callout truly stays suppressed on the staff SVG.
9. **Hover ghost on touch:** confirm no stuck ghost note/cell after a tap on iOS (pointerleave ordering).
10. **Postcard render** in all three themes — canvas text metrics and theme colors only verifiable by eye.
