## Done this session
- **Mobile grid fix (`d5cee18`).** The Grid editor was invisible on phones — pitch labels crammed into a sliver over the octave/Randomize controls. Cause: on the ≤720px stacked layout the regions size to content and the page scrolls, so no ancestor hands the grid a *definite* height; its `grid-template-rows: repeat(N, 1fr)` rows then collapse to ~0. (The Staff surface was fine because its SVG has an intrinsic height.) Fix, scoped to the same ≤720px breakpoint so desktop is untouched: `minmax(24px, 1fr)` row floor + let the editor size to content (`height:auto`, `flex:0 0 auto`) instead of being clipped by the constrained `.staff-frame`. Verified numerically across In Key (15 rows), Chromatic (25 rows), desktop (unchanged).
